Beginner Guide — Full Setup Track (4–8 Hours)

Complete this end-to-end and you’ll finish with a stable OpenClaw setup, a working memory loop, and one repeatable workflow.

Phase 0 — Prep (15–30 min)

  1. Reserve a focused time block (minimum 2 hours).
  2. Create your workspace folder.
  3. Open a setup log note (decisions, failures, fixes).

Exit criteria: time block + workspace + setup log ready.

Phase 1 — Verify CLI Access (20–40 min)

openclaw --help

If command fails, install OpenClaw first and retry. Do not continue with a broken baseline.

Phase 2 — Gateway Health (20–30 min)

openclaw gateway start
openclaw gateway status

If unhealthy:

openclaw gateway restart
openclaw gateway status

Exit criteria: healthy status confirmed and logged.

Phase 3 — Build Core Files (45–90 min)

  • SOUL.md — behavior and tone
  • USER.md — preferences and priorities
  • AGENTS.md — operating rules and approval boundaries
  • TOOLS.md — local environment notes
  • MEMORY.md — long-term distilled context
  • memory/YYYY-MM-DD.md — daily session record

Minimum quality bar: 5–10 meaningful lines per file, not placeholders.

Phase 4 — First End-to-End Workflow (60–120 min)

  1. Select one high-value task.
  2. Define expected output before starting.
  3. Execute with OpenClaw.
  4. Save result to a versioned file.
  5. Run one revision pass.

Examples: SOP draft, project plan, issue triage report, checklist system.

Phase 5 — Memory and Reliability Loop (45–60 min)

  1. Log Wins/Decisions/Open Loops/Next Actions in daily memory.
  2. Promote durable facts to MEMORY.md.
  3. Create weekly review reminder.

Use templates from Library.

Troubleshooting

  • Command not found: reopen terminal and verify installation path.
  • Gateway unhealthy: restart and validate status before proceeding.
  • Behavior inconsistency: tighten identity and boundaries in core files.
  • Forgets context: enforce daily memory updates and weekly distillation.

Need deeper diagnosis? Open Troubleshooting Atlas.

Completion Checklist

  • [ ] OpenClaw command works
  • [ ] Gateway healthy
  • [ ] Core files written with real content
  • [ ] One real workflow completed
  • [ ] Daily + long-term memory loop active

When complete, continue to the Advanced Guide.
