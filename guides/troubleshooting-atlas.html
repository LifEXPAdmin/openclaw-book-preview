Troubleshooting Atlas

Use this matrix during execution: identify the symptom, test the likely root cause, apply smallest fix, validate, and log the lesson.

Core Recovery Loop

  1. Identify failing layer (model/instruction/memory/tool/workflow).
  2. Contain impact (pause external actions, avoid compounding edits).
  3. Apply smallest fix.
  4. Validate with one explicit test.
  5. Record lesson and prevention step.

Symptom → Likely Cause → Fix

SymptomLikely CauseFix
Assistant behavior drifts day-to-dayIdentity files vague or staleTighten SOUL/USER/AGENTS and add concrete boundaries
Context forgotten between sessionsMemory files not updated consistentlyEnforce daily log + weekly distillation SOP
Long tasks stall with no visibilityNo milestone update contractAdd COMPLETED/CURRENT/NEXT+ETA check-ins
Unsafe external action riskNo pre-flight safety gateRequire request/accuracy/privacy/channel verification before send
Delegated output low qualityAmbiguous task briefUse delegation template with acceptance criteria and output format
Too many tools, low reliabilityNo tool-choice rubricPrefer first-class tools; reserve shell for bounded fallback
