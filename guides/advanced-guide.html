Advanced Guide — Operator Track (6–12 Hours)

Design for reliability under complexity: architecture, delegation quality, observability, and throughput controls.

Module 1 — Architecture Design (60–90 min)

  1. Document instruction layer (identity + policy files).
  2. Document state layer (memory storage + hygiene cycle).
  3. Document execution layer (tool-selection + session topology).
  4. Document governance layer (approval boundaries + audit trail).

Deliverable: one-page architecture map with owners and review cadence.

Module 2 — Delegation Framework (90–120 min)

Create and test a standard delegation brief requiring:

  • Objective + scope + non-goals
  • Constraints and prohibited actions
  • Acceptance criteria
  • Output format and merge-back checklist

Validation: run one delegated task and score output against criteria.

Module 3 — Memory Governance (60–90 min)

  1. Adopt daily logging rubric.
  2. Define weekly distillation SOP.
  3. Define stale-memory cleanup criteria.

Why: high-signal memory improves decision quality and reduces drift.

Module 4 — Reliability Engineering (90–120 min)

For your top 3 workflows, create:

  • Failure modes list
  • Detection signals
  • Recovery playbook
  • Postmortem template

Reference matrix: Troubleshooting Atlas.

Module 5 — Throughput + Quality Controls (90–150 min)

  1. Baseline one workflow (time + quality score).
  2. Parallelize only independent work chunks.
  3. Add checkpoints at major milestones.
  4. Track defects and rework causes.

Rule: throughput without quality gates creates expensive chaos.

Advanced Completion Criteria

  • [ ] Architecture map documented
  • [ ] Delegation template proven in at least one run
  • [ ] Memory governance cadence active
  • [ ] Recovery playbooks for top workflows
  • [ ] Throughput and quality metrics tracked

If all boxes are checked, your setup is operational—not ad hoc.
